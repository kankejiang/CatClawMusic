## 跳过已有模式补齐缺失元数据 (23:30)

### 用户反馈
"一键匹配 再勾选跳过已有的情况会补齐缺失信息，比如说已经有封面了但是没有歌手信息即使勾选了跳过已有也会补齐歌手信息"

### 根因分析
`NetEaseMusicScraper.SearchArtistsAsync` 返回的结果里 `Gender = null`、`Region = null`，只有 `Description` 有值。
而 `AutoMatchAsync` 网易云分支只调了 `SearchArtistsAsync`，没有调能返回地区/性别的 `GetArtistInfoAsync`。

### 修复内容

1. **过滤逻辑**（`ArtistMatchFragment.cs`）
   - "跳过已有"模式下，`Birthday` 也纳入过滤条件
   - 现在只要有任一勾选字段缺失，艺术家就会被纳入匹配列表

2. **`NetEaseMusicScraper.ArtistInfo` 类**
   - 新增 `Gender`、`Birthday` 字段

3. **`GetArtistInfoAsync` 方法**
   - 正确解析 `area` 字段（数字类型），映射为地区名：1=中国、2=日本、3=韩国、4=欧美
   - 新增解析 `gender`/`sex`（性别）、`birthday`/`birth`（生日）

4. **`AutoMatchAsync` 网易云分支**
   - 搜索后额外调用 `GetArtistInfoAsync` 获取详细信息
   - 将 `ArtistInfo` 的 `Gender`、`Country`、`Description`、`Birthday` 合并到 `bestMatch`

5. **`AutoMatchMultiSourceAsync` 多来源分支**
   - 如果 `bestMatch` 来自网易云，同样额外调用 `GetArtistInfoAsync` 补齐元数据

### 实现内容

1. **新建 `MusicBrainzScraper.cs`**（`CatClawMusic.Data/`）
   - 调用 MusicBrainz REST API：`https://musicbrainz.org/ws/2/artist/?query=name&fmt=json`

   - 返回字段：Gender（男/女）、Region（地区）、Birthday（生日）、Description（简介）

2. **新建 `TheAudioDbScraper.cs`**（`CatClawMusic.Data/`）
   - 调用 TheAudioDB API：`https://www.theaudiodb.com/api/v1/json/123/search.php?s=name`
   - 免费 Key：`123`（硬编码）
   - 返回字段：Gender（男/女）、Region（国家）、Birthday（出生年份）、Description（中文简介）、CoverUrl（封面）
   - 支持下载封面到缓存目录

3. **`MainApplication.cs`** — 注册两个新 Scraper
   - 添加 `MusicBrainzScraper` 和 `TheAudioDbScraper` 的 singleton 注册
   - 注册到 `IArtistMetadataScraper` 服务集合（自动加入多来源并行搜索）

4. **两个 Fragment** — 添加来源选项
   - `ArtistMatchFragment.cs` 和 `ArtistMatchDetailFragment.cs` 的 `SourceChipToName` 字典
   - 添加 `"chip_musicbrainz" = "MusicBrainz"` 和 `"chip_theaudiodb" = "TheAudioDB"`

5. **两个 XML 布局** — 添加 Chip
   - `fragment_artist_match.xml` 和 `fragment_artist_match_detail.xml`
