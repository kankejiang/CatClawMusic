### 修复的编译错误

1. **错误：`BaiduBaikeScraper` 不实现接口成员 `SourceName`**
   - 原因：之前用 `public const string SourceName = "百度百科";`（字段），但接口要求属性
   - 修复：改为 `public string SourceName => "百度百科";`（属性）

2. **错误：`WebUtility` 不存在**
   - 原因：`WebUtility.HtmlDecode` 需要 `System.Net` 命名空间
   - 修复：添加 `using System.Net;`

3. **错误：`BaikeEntry` 构造函数参数不匹配**
   - 原因：`BaikeEntry` 是 record，但代码中使用 `new BaikeEntry(string, string)` 时参数名称不匹配
   - 修复：重写 `BaiduBaikeScraper.cs`，使用正确的 record 语法 `private record BaikeEntry(string Title, string Url);`

4. **错误：`ExtraInfo` 是属性，不能用 `ref` 传递**
   - 原因：`ArtistSearchResult.ExtraInfo` 是属性，C# 不允许 `ref` 到属性
   - 修复：在 `ParseInfobox` 中使用局部变量 `var extraInfo = "";` 代替 `ref`

5. **错误：switch 语句包含多个相同标签（"艺名"和"星座"）**
   - 原因：代码中 `case "艺名"` 出现了两次（一次是"艺名"，一次是"艺名"的 Unicode normalization 问题）
   - 修复：重写 `ParseInfobox` 的 switch 语句，确保每个 label 只出现一次

6. **错误：缺少 `List<string>` 的 using 引用**
   - 原因：`ArtistDetailFragment.cs` 中使用了 `new List<string>()` 但没有 `System.Collections.Generic` 引用
   - 修复：添加 `using System.Collections.Generic;`

## 修复歌词四项问题 (00:15)

### 用户反馈的四个问题
1. 外挂的 `.ttml` 歌词无法识别，显示"暂无歌词"
2. TTML 歌词在两个人一起合唱的时候会卡死
3. 对唱歌词的布局和普通 LRC 歌词的布局要改成左右布局（根据 TTML/AMLL 内的内容布局）
4. 扫描歌曲后清理删除的歌曲后会闪退

### 实现内容

#### 1. 修复外挂 `.ttml` 歌词无法识别
文件：`CatClawMusic.Core/Services/LyricsService.cs`

- `GetLocalLyricsAsync` 方法：当 `song.LyricsPath` 是 `content://` URI 时，原来只调用 `ParseLrc`，现在改为异步按 LRC → TTML → AMLL → 纯文本顺序解析

#### 2. 修复 TTML 歌词合唱时卡死
文件：`CatClawMusic.Core/Services/LyricsService.cs`

- 新增 `ParseTtmlAsync` 方法：包装 `ParseTtml` 在 `Task.Run` 中避免阻塞 UI 线程
- 更新 `preferEmbedded` 分支：改用异步解析 TTML

#### 3. 实现对唱左右布局（根据 TTML/AMLL 的 role 属性）

**模型层：**
文件：`CatClawMusic.Core/Models/Lyrics.cs`
- `LrcLyricLine` 新增 `Alignment` 字段（0=左对齐，1=居中，2=右对齐）
- `LrcLyrics` 新增 `HasPerLineAlignment` 字段（标记是否有逐行对齐方式）

**解析层：**
文件：`CatClawMusic.Core/Services/LyricsService.cs`
- `ParseTtml`：解析 `<p>` 元素的 `role` 属性，设置 `Alignment` 字段
- 新增 `ParseTtmlAlignment` 方法：解析 TTML `role` 属性（支持 `itunes:role`、自定义 `role` 属性、父级 `<div>` 的 `role`）
- `ParseAmll`：实现 `ParseAmllRole` 方法，解析 AMLL JSON 的 `role` 字段
- 在 `ParseTtml` 和 `ParseAmll` 中设置 `HasPerLineAlignment = lines.Any(l => l.Alignment != 1)`

**UI 层：**
文件：`CatClawMusic.UI/Fragments/FullLyricsFragment.cs`
- 新增 `GetGravityFromAlignment(int alignment)` 方法：将对齐值转换为 `GravityFlags`
- 更新 `RebuildLyrics()` 方法：如果 `lyrics.HasPerLineAlignment` 为 true，使用每行的 `Alignment` 字段；否则使用全局 `_lyricAlignment` 设置

#### 4. 修复清理删除歌曲后闪退

文件：`CatClawMusic.UI/ViewModels/SettingsViewModel.cs`
- `CleanupDeletedFolderSongsAsync`：原来只收集 SAF URI，现在也收集自建文件管理器添加的文件夹路径（`ScanSettings.GetLocalFolderPaths()`）

文件：`CatClawMusic.Data/MusicDatabase.cs`
- `RemoveStaleSongsAsync`：原来用 `retainPaths.Contains(s.FilePath)` 精确匹配，现在改用 `retainPaths.Any(p => s.FilePath.StartsWith(p, ...))` 前缀匹配（因为 `retainPaths` 是文件夹路径，`s.FilePath` 是文件路径）
