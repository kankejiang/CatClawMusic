# 2025-06-14

## 多艺术家支持 - SongArtists 多对多关联

将 Song→Artist 从 1:1（单 ArtistId）升级为 多对多（SongArtists 中间表）。

### 变更文件
- **新增**: `CatClawMusic.Core/Models/SongArtist.cs` — 中间表模型 (SongId + ArtistId)
- **修改**: `CatClawMusic.Core/Models/Song.cs` — 新增 `AllArtists` [Ignore] 运行时字段
- **修改**: `CatClawMusic.Data/MusicDatabase.cs` — 创建 SongArtists 表、索引、迁移、多艺术家查询、批量保存、AllArtists 填充
- **修改**: `CatClawMusic.Data/MusicScanner.cs` — 批量插入后创建 SongArtist 关联（保存完整拆分列表）
- **修改**: `CatClawMusic.Core/Services/TagReader.cs` — 从 TagLib 读取全部 Performers（`string.Join(" / ", tag.Performers)`）

### 核心逻辑
- `MusicUtility.SplitArtistNames("周杰伦/林俊杰")` → ["周杰伦", "林俊杰"]
- 扫描入库时：第一个作为主 ArtistId，全部写入 SongArtists
- 查询时：`SearchSongsAsync` / `GetSongsByArtistAsync` 通过 `LEFT JOIN SongArtists` 匹配任一艺术家
- 已有数据迁移：`MigrateToMultiArtistAsync()` 在首次初始化时将现有 ArtistId 写入 SongArtists
- 删除时级联清理：`RemoveStaleSongsAsync` / `ClearLocalSongsAsync` 等均清理 SongArtists

### 设计原则
- 保持 Song.ArtistId 向后兼容（主艺术家）
- AllArtists 字段仅在需要时批量填充（`GetAllArtistsForSongsAsync`）
- 乐队名保护机制仍然有效（"AC/DC" 等已知乐队不拆分）

## 修复：重复刷新音乐库导致清空 0 首的问题

**Bug 原因（两处）**：

1. **UI 不显示已有歌曲**：`InsertSongsBatchAsync` 只把**新增**歌曲传回回调，已有歌曲只做 UPDATE 不会触发回调。刷新时 `Songs.Clear()` 后，已有歌曲永远不会被加回 UI。

2. **数据库误删**：`RemoveStaleSongsAsync` 用扫描发现的路径对比数据库路径，如果扫描策略变化导致路径格式不一致（SAF `content://` vs 文件系统 `/storage/emulated/0/`），所有数据库记录会被孤立删除。

**修复**：

1. `BackgroundScanAsync` 扫描完成后，从数据库全量加载全部本地歌曲重新填充 UI（而非依赖增量回调）
2. `RemoveStaleSongsAsync` 调用前检查 `scannedPaths.Count > 0`，为空时跳过清理避免误删

修改文件：`CatClawMusic.UI/ViewModels/LibraryViewModel.cs`

## 修复：探索页艺术家 Tab 显示合并艺术家名（如 "国风堂/哦漏"）

**问题**：探索页艺术家列表显示 "国风堂/哦漏" 作为一个艺术家，而不是拆分为两个。

**根因**：
1. 历史扫描中，Artist 表可能存有合并名称（如 `Name="国风堂/哦漏"`）
2. `GetArtistsWithSongCountAsync` 只按 `s.ArtistId`（主艺术家）分组计数，未通过 SongArtists 多对多表统计

**修复**：
1. `MusicDatabase.SplitCombinedArtistsAsync()`：自动迁移，查找名称含 " / " 的艺术家 → 拆分名称 → 创建/查找独立艺术家 → 更新 Song.ArtistId + SongArtists 关联 → 删除旧合并记录
2. `ExploreDataService.GetArtistsWithSongCountAsync()`：改为通过 SongArtists 多对多表补充计数合作歌曲的次要艺术家
3. `MusicDatabase.QuerySongArtistsBySongIdsAsync()`：新增批量查询方法

修改文件：
- `CatClawMusic.Data/MusicDatabase.cs`（迁移 + 查询方法）
- `CatClawMusic.Data/ExploreDataService.cs`（计数逻辑）

## 防御性修复：探索页艺术家 Tab 过滤合并艺术家名

**问题**：数据库迁移可能静默失败，导致 "国风堂/哦漏" 仍显示为合并艺术家。

**追加修复**：`ExploreDataService.GetArtistsWithSongCountAsync()` 增加 `.Where(a => !IsCombinedArtistName(a.Name))` 过滤，通过 `SplitArtistNames` 验证名称能否拆分，能拆出多个名字的合并艺术家直接跳过不显示。

修改文件：`CatClawMusic.Data/ExploreDataService.cs`
- 新增 `IsCombinedArtistName()` 辅助方法
