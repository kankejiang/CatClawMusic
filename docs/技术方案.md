# 猫爪音乐 — 技术方案

## 1. 概述

**猫爪音乐** 是基于 .NET 9 MAUI/Xamarin.Android 的 Android 跨平台音乐播放器，支持本地和网络播放。

### 核心原则

- MVVM 架构，页面与逻辑分离
- 接口驱动设计，层间依赖反转
- SQLite 本地持久化 + TagLibSharp 元数据
- AndroidX Media3 ExoPlayer 音频引擎
- 少女可爱风浅色 UI，锁定 Light 模式
- 桌面悬浮歌词（全厂商 ROM 适配）

## 2. 系统架构

### 2.1 分层结构

```
Presentation Layer (UI)
    Pages (.xaml + .xaml.cs)
    ViewModels (BindableObject)
    Platforms (Android 特定实现)
        ↓ 依赖
Business Logic Layer (Core)
    Interfaces (IAudioPlayerService, IMusicLibraryService, ...)
    Services (PlayQueue, LyricsService, TagReader, MusicUtility)
    Models (Song, Album, Playlist, ConnectionProfile, ...)
        ↑ 被依赖
Data Access Layer (Data)
    MusicDatabase (SQLite 上下文)
    MusicLibraryService (扫描 + DB 存储)
    SubsonicService (Navidrome API 客户端)
    WebDavService (WebDAV 协议)
    NetworkMusicService (协议路由工厂)
```

### 2.2 依赖注入 (MauiProgram.cs)

```
Singleton（整个应用生命周期内单例）:
  - MusicDatabase(dbPath)
  - ISubsonicService → SubsonicService
  - INetworkFileService → WebDavService
  - INetworkMusicService → NetworkMusicService
  - IAudioPlayerService → Platforms.Android.AudioPlayerService
  - ILyricsService → LyricsService
  - IMusicLibraryService → MusicLibraryService
  - IPermissionService → Platforms.Android.PermissionService
  - PlayQueue

Transient（每次请求新建）:
  - 7 个 ViewModel (Library, NowPlaying, Settings, Search, Playlist, WebDav, Navidrome)
  - 7 个 Page (Library, NowPlaying, Playlist, Search, Settings, WebDav, Navidrome)
```

## 3. 核心模块设计

### 3.1 音频播放 (AudioPlayerService)

```
封装 AndroidX.Media3.ExoPlayer
- PlayAsync(path) → Reset → SetDataSource → PrepareAsync → Start
- PauseAsync/StopAsync/SeekAsync
- Volume (get/set, 映射到 MediaPlayer.SetVolume)
- 500ms Timer 推送 PositionChanged 事件
- StateChanged: Playing/Paused/Stopped/Error/Buffering
- IDisposable: Release + Dispose
- 支持 content:// URI 原生播放（无需 ParcelFileDescriptor 绕路）
```

### 3.2 播放队列 (PlayQueue)

```
4 种模式:
  Sequential     → 顺序播放，到末尾停止
  Shuffle        → Fisher-Yates 洗牌，到末尾重洗
  SingleRepeat   → 单曲循环
  ListRepeat     → 列表循环

核心方法:
  SetSongs(IEnumerable<Song>)  → 重置队列
  Next() / Previous()          → 历史栈导航
  SelectSong(songId)           → 手动选歌
```

### 3.3 歌词解析 (LyricsService)

```
多源查找优先级:
  1. 音频文件嵌入歌词 (TagLib.Tag.Lyrics)
  2. 同名 .lrc 文件 (song.mp3 → song.lrc)
  3. 同目录模糊匹配 .lrc 文件
  4. 网络歌词提供者（预留插件接口）

解析细节:
  - 正则: [(\d+):(\d+)(?:\.(\d+))?]
  - 支持 [mm:ss.xx]、[mm:ss.xxx]、[mm:ss]
  - 元数据: ti/ar/al/by/re/ve
  - DateTime.TryParse 时间戳有效性校验
  - 跳过注释行(//)和纯音乐标记
```

### 3.4 音乐扫描 (MusicLibraryService + TagReader)

```
扫描路径:
  /storage/emulated/0/Music
  /storage/emulated/0/Download
  SD卡 /Music (如果有)
  /猫爪音乐 / /方糖音乐

TagLibSharp 读取字段:
  Tag.Title → Song.Title
  Tag.FirstPerformer / FirstAlbumArtist → Song.Artist
  Tag.Album → Song.Album
  Properties.Duration.TotalSeconds → Song.Duration
  Properties.AudioBitrate → Song.Bitrate
  Tag.Pictures[0].Data → Album Cover (byte[])
  Tag.Lyrics → Embedded Lyrics (string)
```

### 3.5 Navidrome / Subsonic API

```
认证: MD5(password + salt) token
协议版本: 1.16.1
端点:
  - rest/ping                    → 测试连接
  - rest/search3?query=xxx       → 搜索
  - rest/getRandomSongs?size=500 → 浏览音乐库
  - rest/getAlbumList2?type=newest → 专辑列表
  - rest/stream?id=xxx           → 流媒体播放
  - rest/getCoverArt?id=xxx      → 专辑封面
  - rest/getLyrics?id=xxx        → 歌词

JsonDocument 解析，结果映射到 Core.Models.Song/Album
```

### 3.6 桌面歌词 (DesktopLyricService)

```
Android WindowManager 悬浮窗实现，WindowManagerLayoutParams.Type = ApplicationOverlay/Phone

核心功能:
  - 实时歌词同步：监听 NowPlayingViewModel.CurrentLyricLine + PositionChanged
  - KTV 双行模式：左上亮色(当前行 20sp) + 右下暗色(下一行 15sp)，对角线滚动
  - 拖拽：仅支持上下移动（X=0, MatchParent 宽度），手指按下时背景半透明
  - 锁定：🔒 按钮切换，锁定后不可拖动，手指离开 2 秒后自动隐藏图标
  - 外观自定义：字体大小(12-36sp)、颜色(10种预设)、粗体、背景透明度(0-100%)、窗口边框开关
  - 设置持久化：SharedPreferences 存储位置、字体、颜色、模式

全厂商 ROM 悬浮窗权限适配:
  小米/HyperOS → MIUI安全中心 AppPermissionsEditorActivity + MIUI_FLOATING_WINDOW 权限
  华为/EMUI   → AppOpsManagerEx.checkHwOpNoThrow + AddViewMonitorActivity
  荣耀/MagicOS → hihonor.systemmanager（fallback 华为方案）
  OPPO/ColorOS → com.oppo.safe FloatingWindowListActivity
  vivo/OriginOS → SoftPermissionDetailActivity
  魅族/Flyme  → SHOW_APPSEC Intent
  通用         → Settings.ACTION_MANAGE_OVERLAY_PERMISSION

AppOps 双权限栈检测:
  Settings.CanDrawOverlays() (AOSP层) + AppOpsManager.checkOpNoThrow(OP_SYSTEM_ALERT_WINDOW) (厂商层)
```

### 3.7 权限管理 (PermissionService)

```
Android 版本适配:
  ≥ 13 (Tiramisu)  → READ_MEDIA_AUDIO
  6-12              → READ_EXTERNAL_STORAGE
  < 6               → 无需运行时权限

悬浮窗权限 (SYSTEM_ALERT_WINDOW):
  Settings.CanDrawOverlays() + AppOps 反射检测
  厂商 ROM 专项适配：小米/华为/荣耀/OPPO/vivo/魅族
  自动跳转厂商专属权限管理页面

UI 流程:
  检查权限 → 未授权显示"授予权限"卡片
  → 点击调用系统弹窗 → 授权后自动扫描
```

### 3.7 数据库 (MusicDatabase)

```
SQLiteAsyncConnection, 8 张表:
  Song / Album / Playlist / PlaylistSong /
  PlaybackStats / RecentPlay / CachedSong / ConnectionProfile

索引:
  idx_songs_artist / idx_songs_album
  idx_lastplayed (DESC) / idx_playcount (DESC)
```

## 5. 网络协议扩展

`ProtocolType` 枚举预留了 SMB(2)、DLNA(3)、FTP(4)、NFS(5)。

添加新协议只需:
1. 实现对应的 `IXxxService` 接口
2. 在 `NetworkMusicService.ScanAsync()` 的 switch 中添加 case
3. 创建设置二级页面 (XAML + ViewModel)
4. 注册到 MauiProgram DI 和 AppShell Route
